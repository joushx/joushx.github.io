Entwurf für HTTP 2.0
Dieser Artikel ist älter als zwei Jahre und womöglich veraltet!

Die IETF hat jetzt einen ersten benutzbaren Entwurf für den Nachfolger von HTTP 1.1 vorgelegt.

Im neuen Protokoll HTTP/2.0, das vormals auf den Namen SPDY hörte und von Google entwickelt wurde, sind eine Reihe von nützlichen Neuheiten enthalten, um HTTP auf den neuesten Stand seit der Veröffentlichung von Version 1.1 im Jahr 1999 zu bringen.

Die Übertragung soll dadurch beschleunigt und optimiert werden. Dies wird unter anderem durch das Multiplexen der Verbindung, also dem Senden von mehreren Dokumenten über eine TCP-Verbindung, erreicht.

Die Notwendigkeit einer Verschlüsselung für alle Verbindungen wurde jedoch gegenüber SPDY in dieser Version des Entwurfs fallen gelassen.

Auf jeden Fall freue ich mich schon auf den Produktiveinsatz des Protokolls!
